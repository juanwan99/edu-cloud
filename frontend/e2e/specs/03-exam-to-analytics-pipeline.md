# 考试→分析 关联测试（长链路）

这是一个关联测试，模拟完整业务流程：考试列表 → 考试详情 → 成绩分析报告。
各 Phase 共享浏览器会话，前一步的数据传递给后一步。

## Phase 1: 登录 + 角色准备

### 1.1 登录
- 登录 admin/123456
- 截图: phase1-login

### 1.2 切换到学校角色
- 切换角色到学校管理员（如果需要）
- 验证: 侧边栏显示学校级菜单
- 截图: phase1-school-role

## Phase 2: 考试列表

### 2.1 进入考试管理
- 点击侧边栏"考试管理"
- 验证: 显示考试列表
- 截图: phase2-exam-list

### 2.2 确认有考试数据
- 验证: 列表中至少有 1 条考试记录
- 记录: 第一条考试的名称和 ID（后续 Phase 使用）
- 截图: phase2-exam-data

## Phase 3: 成绩分析

### 3.1 进入成绩分析
- 点击侧边栏"成绩分析"
- 验证: 显示分析页面
- 截图: phase3-analytics-entry

### 3.2 选择考试
- 在分析页面的筛选器中选择 Phase 2 记录的考试
- 验证: 页面显示该考试的分析数据
- 截图: phase3-analytics-data

### 3.3 数据完整性
- 验证: 页面包含以下至少 3 项分析维度:
  - 成绩分布/分数段
  - 班级对比
  - 学科分析
  - 知识点分析
  - 排名数据
- 截图: phase3-analytics-detail

## Phase 4: 学生画像（从分析页跳转）

### 4.1 点击学生
- 如果分析页面有学生列表/排名，点击某个学生名字
- 如果没有，从侧边栏进入"学生画像"
- 验证: 进入学生画像页面
- 截图: phase4-student-profile

### 4.2 画像数据
- 验证: 页面包含学生基本信息
- 验证: 页面包含成绩趋势图或数据
- 截图: phase4-profile-data

## 预期结果
- 全链路 4 个 Phase 跑通
- 考试数据 → 分析报告 → 学生画像 数据流闭环
- 无空白页/无数据/500 错误

# Fix Intent Card — F002 & F003

## F002: Class-scope 守卫

**root_cause**: conduct 管理端端点只校验"角色有权限"，不校验"路径中的 class_id 属于当前角色的 class_ids"，也不校验"路径中的 {resource_id} 属于 class_id"。

**preserved_invariants**:
- ORC-001 管理端所有 `/{class_id}/...` 路径必须验证 class_id ∈ visible_class_ids
- ORC-002 所有嵌套资源（rule_category, rule_item, group, semester, record）的 class_id 必须等于路径中的 class_id
- ORC-003 跨班资源 ID 不可在本班路径下被修改/删除
- ORC-004 platform_admin/district_admin/principal/academic_director 可访问全部班级（get_visible_class_ids 返回 None）

**non_goals**:
- 不改变现有权限定义（VIEW/MANAGE/MANAGE_RULES/MANAGE_PARENTS/EXPORT）
- 不改变 get_visible_class_ids 的行为
- 不改变家长端路由（家长端另行通过 guardian_student_links 校验）

**allowed_change_surface**:
- `src/edu_cloud/modules/conduct/permissions.py` — 新增 `check_class_scope()` 和 `check_resource_class()` 辅助
- `src/edu_cloud/modules/conduct/admin_router.py` — 所有 `/classes/{class_id}/...` 端点补 class_scope 校验
- `src/edu_cloud/modules/conduct/admin_service.py` + `rules_service.py` — 嵌套资源写操作前补 class_id 归属校验
- `tests/test_conduct/test_admin_api.py` — 补跨班越权的红测

**verification**:
- 新增红测：homeroom_teacher(class_A) 访问 /classes/B/config → 403
- 新增红测：homeroom_teacher 用外班 rule_item_id 更新本班规则 → 404
- 原有 78 tests 不回归

## F003: Agent 工具 DataScope

**root_cause**: 6 个 conduct Agent 工具直接信任 `input["class_id"]/["student_id"]`，不做 scope 交集。`ctx.class_ids` 存在但未使用。

**preserved_invariants**:
- ORC-005 AI 工具读端：class_id ∈ ctx.class_ids（若 ctx.class_ids 非 None）
- ORC-006 AI 工具写端（add_conduct_points）：class_id 必须在 scope 内
- ORC-007 student 敏感度工具：student.class_id ∈ ctx.class_ids
- ORC-008 parent 角色通过 guardian_student_links 限定

**non_goals**:
- 不改变 ToolContext 接口
- 不改变工具注册元数据（domain/risk_level/allowed_roles）
- 不引入新的 scope 模型（复用 ctx.class_ids）

**allowed_change_surface**:
- `src/edu_cloud/ai/tools/conduct.py` — 6 个工具内部补 scope 校验
- `tests/test_conduct/test_agent_tools.py` — 补 scope 违反的红测

**verification**:
- 新增红测：ctx.class_ids=["A"]，调用 get_conduct_rankings(class_id="B") → ToolResult.ok=False, error="out of scope"
- 新增红测：add_conduct_points 写外班 → 拒绝

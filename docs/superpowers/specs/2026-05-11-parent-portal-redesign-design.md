# 家长端 UI 系统性优化设计文档

> 方案代号：「深墨金」品牌强化
> 定位：独立移动端产品
> 日期：2026-05-11

## 1. 设计定位

家长端是面向家长的**独立移动产品**，不是主站的移动适配版。与主站共享「冷暖撞色」品牌色板（深墨 + 金黄 + 冷紫 + 橙），但拥有独立的视觉语言和交互体验。

核心设计理念：**家长信心 App**，不是学校数据控制台。首屏回答"今晚我需要关注什么？"，而非平铺 10 个数据卡片。

### 2.3 切换机制

## 4. 导航与布局

### 4.1 顶部栏（56px）

- 背景：`var(--surface-1)` + 底部 `1px var(--border)`
- 左侧：孩子头像（`28px` 圆形）+ 姓名 + 下拉箭头 → 底部抽屉切换孩子
- 姓名下方：班级名，`--fs-label` + `--text-3`
- 右侧：Lucide `Bell` 图标，未读时金色圆点
- 多孩子切换：底部抽屉（`--surface-raised`），选中态金黄高亮

### 4.2 底部 Tab 栏（64px + safe-area）

4 个 Tab：

| Tab | 图标（Lucide） | 标签 |
|-----|---------------|------|
| 首页 | `Home` | 首页 |
| 成绩 | `BarChart3` | 成绩 |
| 表现 | `Star` | 表现 |
| 我的 | `UserRound` | 我的 |

- 背景：`var(--surface-1)`，顶部 `1px var(--border)`
- 图标 `24px`，未选中 `--text-3`，选中 `--color-accent`
- 标签 `--fs-tab`（`11px`），跟随图标颜色
- `padding-bottom: env(safe-area-inset-bottom)`

### 4.3 页面过渡

| 场景 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| Tab 切换 | `fade` | `150ms` | `ease` |
| 进入详情 | `slide-left` | `220ms` | `cubic-bezier(.2,0,0,1)` |
| 返回 | `slide-right` | `220ms` | `cubic-bezier(.2,0,0,1)` |
| 底部抽屉 | `slide-up` | `240ms` | `cubic-bezier(.2,0,0,1)` |

### 4.4 页面通用结构

- 内容区水平 padding `var(--space-6)`（`24px`）
- 卡片间距 `var(--space-5)`（`20px`）
- 底部留白 `80px`（Tab 栏空间）
- 下拉刷新：仅首页、成绩、表现三个数据页

### 4.5 加载策略

| 场景 | 方式 |
|------|------|
| 首次进入 / 切换孩子 | 骨架屏（shimmer `1.5s` 循环，最多 `3s`） |
| 下拉刷新 | 顶部金黄 spinner → ✓ 停留 `400ms` |
| 单个操作 | 按钮内 spinner |
| 超过 10s | 进度提示文字 |

## 5. 页面信息架构

### 5.1 首页（行动导向）

从上到下：

1. **今晚关注卡**（`--color-accent-surface` 背景）
   - 有行动项：`⚡ 今晚需关注 1 项` + 具体建议 + 跳转
   - 无事项：`✓ 今日无需关注事项`（安心提示）
2. **学业趋势卡**
   - sparkline 折线（近 5 次考试）
   - 班级位置（百分位 `前 35%`）+ 变化方向
   - 最近一次考试名称和总分
3. **本周表现卡**
   - 加分/待改善次数
   - 最近事件横向滚动小卡片
   - 学期累计分数
4. **最新动态**（列表，2-3 条）
   - 从已有数据派生：新成绩发布、新行为记录、排名变化
   - 不需要独立通知 API，前端从 scores/records 接口的时间戳推断

设计要点：
- 负面信息措辞：`待改善` `建议复盘` `需关注`，避免 `退步` `差` `不及格`
- 所有数据卡显示来源和更新时间：`来自 七年级3班 · 21:08 更新`

### 5.2 成绩页

顶部 `NSegmented`：`考试` / `学科`

**考试视图**：
- 最近考试卡：总分 + 班级排名 + 年级排名（三个统计块）
- 学科分数列表：水平进度条 + 班均对比线，低于班均标 `⚠`（`--color-warning`）
- 考试列表按时间倒序，可折叠

**学科视图**：
- 单学科趋势折线图
- 错题概况：未掌握 / 练习中 / 已掌握

### 5.3 表现页

顶部 `NSegmented`：`记录` / `排名` / `班规`

**记录视图**：
- 按日分组时间线
- 正向事件 `--color-success`，负向 `--color-warning`（不用红色）
- 负向记录关联班规条目，可内联展开

**排名视图**：
- 顶部卡：百分位 `前 35%` + 精确名次 + 趋势
- 排名列表：当前孩子行用 `--color-accent-surface` 高亮

**班规视图**：
- 搜索框 + 分类折叠列表（复用现有 ParentRules 逻辑）

### 5.4 我的页

- 头像 + 账户信息
- 已绑定孩子列表 + 管理/绑定入口
- 设置列表：外观模式（三选一）、修改密码、通知设置、关于、退出登录

### 5.5 登录 / 注册 / 绑定

独立于 Tab 壳之外的全屏页面：
- 登录：深墨全屏 + 品牌文字 + 表单 + 金黄主按钮
- 注册：同风格，邀请码输入
- 绑定：引导式步骤 + 进度指示

## 6. 微交互与动效

### 6.1 按压反馈

| 元素 | 效果 | 时长 |
|------|------|------|
| 按钮 | `scale(0.97)` | `100ms` |
| 卡片 | 背景升一级 | `120ms` |
| Tab 图标 | `translateY(-1px)` + 颜色切换 | `150ms` |
| 列表项 | `rgba(255,255,255,.04)` 背景 | `80ms` |

### 6.2 骨架屏

- 形状匹配真实卡片布局
- shimmer：`linear-gradient` 左→右，`1.5s` 循环
- 颜色：`var(--surface-2)` 底 + `var(--surface-3)` 高光
- 最多 `3s`，超时变文字提示

### 6.3 下拉刷新

- 触发距离 `60px`
- 金黄 spinner → ✓ 停留 `400ms` → 收回
- 刷新后显示 `更新于 21:08`，`3s` 后淡出

### 6.4 数字滚动

- `--fs-hero` 级别大数字在切换孩子/刷新时从旧值过渡到新值
- `400ms`，`ease-out`

### 6.5 空状态

| 场景 | 内容 |
|------|------|
| 未绑定孩子 | 简笔线条插画 + "绑定孩子后即可查看" + 金黄 CTA |
| 暂无成绩 | "还没有考试记录" + 柔和图标 |
| 暂无表现 | "本周还没有记录，继续加油" |
| 网络错误 | "网络不太给力" + 重试按钮 |

插画风格：简笔线条，`--text-3` 颜色，与暗色主题协调。

### 6.6 通知标记

- 首页 Tab：未读时图标右上角 `6px` 金色圆点
- 顶栏铃铛：未读时 `8px` 金色圆点
- 其他 Tab 不加 badge

## 8. 无障碍

- 正文最小 `15px`，不锁定缩放
- 交互目标 ≥ `48x48px`
- WCAG AA：正文 `4.5:1`，大字/图标 `3:1`
- 不依赖颜色单独传达信息，配合文字/图标/箭头
- 措辞家长友好：`本周表现` `需要关注` `进步` `班级位置`

## 9. 技术实现概要

### 9.1 文件结构变更

- `src/assets/styles/parent-tokens.css`：新增，暗色/亮色双套 token
- `src/layouts/ParentLayout.vue`：重写，主题切换 + 新导航结构
- `src/pages/parent/*.vue`：9 个页面逐页重构
- `src/components/parent/`：新增共享组件（骨架屏、空状态、趋势图等）
- 依赖新增：`lucide-vue-next`

### 9.2 Naive UI 主题覆盖

双层主题策略：
- **页面样式**（卡片、布局、自定义组件）：完全由 `parent-tokens.css` 的 CSS 变量驱动，通过 `[data-theme]` 属性切换暗色/亮色变量集
- **Naive UI 组件内部色**（按钮、输入框、表格等）：仍通过 `NConfigProvider` + `parentThemeOverrides` 覆盖，但 override 对象根据 `data-theme` 动态切换（暗色用自定义 override，亮色用默认 lightTheme + 紫主色 override）

### 9.3 路由变更

- 移除 `/parent/rankings`、`/parent/rules` 独立路由
- `/parent/details` 合并到表现页内部
- 保留：`/parent`（首页）、`/parent/scores`（成绩）、`/parent/conduct`（表现）、`/parent/profile`（我的）
- 登录/注册/绑定路由不变

## 10. 影响范围

| 项目 | 变更 |
|------|------|
| 前端路由 | 减少 2 个顶级路由，Tab 从 5→4 |
| 后端 API | 无变更，复用现有 conduct API |
| 数据库 | 无变更 |
| 依赖 | 新增 `lucide-vue-next` |
| 主站 | 无影响，家长端完全隔离 |

## 核心问题

知识图谱应该如何组织？（不只是可视化问题，是知识模型本身的问题）

**关键约束（用户明确指出）：**
- 知识图谱 ≠ 掌握度图谱。知识图谱是纯粹的知识结构，代表"高中阶段需要掌握哪些知识，以及知识是怎么联系的"
- 掌握度/学情是后续叠加在知识图谱上的数据层，不是知识图谱本身
- 当前实现（力导向图 108 节点 + 335 边）被用户否定："不是给人类看的"
- 质量优先，知识图谱是最关键最核心的内容

## 现有数据

- **108 个 L1 概念**（如"细胞学说""蛋白质""光合作用"），分属 5 个模块（M1-M5）
- **10 个大概念 (BigConcept)** 分组（如"细胞的分子组成""细胞的基本结构"）
- **335 条关系**：prerequisite_hard（前置依赖）、bridge_to（桥接）、contrast（对比）
- **1103 条 evidence**（L0 教材原子事实，作为概念的详情内容）
- 每个概念有：aliases、difficulty(1-5)、bloom_level、review_status、description

## 用户角色

首要用户是教师（教研、备课、审核），其次是 Agent（诊断推荐），再次是家长学生。

## 左侧已有

三级树导航（Module→BigConcept→Concept）已经解决了"课程导航"需求。

## Claude 立场

### 右侧应该展示什么

左侧树回答"知识点在哪"（导航维度），右侧应回答"知识点之间怎么关联"（关系维度）。这是两个正交的维度。

### 组织方式建议

**方案：模块级概念地图（固定布局）**

核心理念：不用力导向物理模拟，而用**固定的、可预测的布局**展示知识结构。

1. **默认视图 = 模块概览**
   - 5 个模块作为 5 个区块（卡片/面板）
   - 每个区块显示模块名 + 大概念数 + 概念数
   - 模块之间用 L2 跨模块关系连线
   - 点击模块 → 进入该模块的概念地图

2. **模块内视图 = 概念地图**
   - 大概念作为分组容器（带标题的区域）
   - 概念作为容器内的节点（名称 + 小图标表示 review_status）
   - 概念之间的 prerequisite_hard 关系用有向箭头连接
   - bridge_to 和 contrast 关系用不同样式的线
   - 布局：大概念按 display_order 排列，概念在组内按 display_order 排列
   - **关键：布局固定，不抖动，教师每次打开看到的是同样的位置**

3. **交互**
   - 点击概念 → 右侧详情抽屉（已有）
   - hover 概念 → 高亮它的直接关系（前置 + 后续）
   - 从树导航点击 → 图谱滚动/聚焦到对应概念

### 为什么不用力导向图

- 教师打开页面看到节点在抖动 → 不专业
- 每次打开布局不同 → 无法形成空间记忆
- 108 个节点全堆一起 → 信息过载
- 力导向的"自组织"对已知结构无意义 — 课程结构是人为设计的，不需要算法"发现"

## 辩论共识（4 轮收敛）

### 知识之间的本质连接（认知科学视角）

| 连接类型 | 认知功能 | 数据对应 | 表达方式 |
|---------|---------|---------|---------|
| 先备依赖 | 认知骨架，学习顺序 | prerequisite_hard (147条) | 驱动主布局（纵向分层） |
| 辅助先备 | 弱依赖，可跳过 | prerequisite_soft (128条) | V1 不进主骨架，选中时弱显示 |
| 跨域桥接 | 知识整合，"原来这两块能连" | bridge_to (31条) | 发现模式，覆盖层 |
| 边界对比 | 防混淆，纠错 | contrast (29条) | 升级为误概念/辨析层 |
| 学习进阶 | 同一概念的掌握深度层级 | sub_concept_progressions（设计有，未实现） | 独立数据层 |
| 跨学科框架 | 结构-功能/物质-能量/稳态-调节 | 缺失 | 教学透镜标签 |

### 三层职责分离

- **左侧（索引控件）**：三级树导航，搜索定位。回答"概念在哪"。保持不变。
- **右侧（理解/行动控件）**：骨架总览 + 关系审查。回答"怎么组织、怎么关联"。本次重设计目标。
- **抽屉（资料仓）**：课标/教材/DA/证据/真题/属性编辑。回答"详细信息"。保持不变。

### 右侧两种模式

**模式 1：骨架总览（默认）** — 分层概念图
- prerequisite_hard 驱动纵向分层（左→右 = 基础→进阶）
- BigConcept 做横向分区（色条标注归属）
- 每个模块 ~20-35 节点，认知负荷可控
- 布局固定，不抖动
- 跨模块前置用徽标标注（如 →M2×3）

**模式 2：焦点透视（选中概念后）**
- 上半：局部路径图（上游 2 层 + 下游 2 层）
- 下半：分组审查表（前置/后继/对比桥接）
- "查看详情"打开已有抽屉

### 关键原则

- 知识图谱 ≠ 掌握度图谱（纯结构 vs 学情叠加）
- 可视化服务模型（模型不丰富时不强行做复杂可视化）
- 放弃一屏展示完整网络（108 节点 + 335 边永远不全铺）
- 列表不能替代图（空间认知有不可替代的价值），但图也不能替代列表（审核需要可读可编辑）
- Agent 在模型之上（消费知识模型，不替代它）

## 外部深度调研结论

### 产品调研

| 产品 | 知识组织 | 启示 |
|------|---------|------|
| ALEKS | Knowledge Space Theory（知识状态建模） | 对 108 概念过重，但"可诊断状态"理念值得借鉴 |
| Khan Academy | 旧 knowledge map 已移除 → skill mastery + learning paths | 图不是终点 |
| IXL | 标准对齐 + 诊断 + action plan | "下一步做什么"比图重要 |
| Knewton Alta | learning objective 图 + 3-hop 回溯补救 | 最接近我们场景 |
| 好未来 | 知识图谱 + 个体学力图谱 | 国内头部也在走"图→模型"路线 |

### 学术信号（2024-2025）

前沿一致指向：不是更炫的图布局，而是认知模型 + 进阶模型 + 学习状态 + LLM 联动。

### 标准框架

- NGSS：三维整合（核心概念 + 科学实践 + 跨学科概念）+ learning progressions
- 1EdTech CASE：competency framework 标准化
- 中国新课标：明确要求"根据大概念来构建课程体系"，核心素养四维度

### 最终判断

对 108 个高中生物概念，最佳实践是：**"课程框架化的概念图 + 进阶模型 + 审查工作流"**。我们的方向被外部证据加强，没有发现被完全忽略的更好范式。

## 1. 目标

将考试生命周期从"基础 5 态"补全为"完整 7 态 + 阅卷任务分配 + 质量监控 + 成绩发布"，使 edu-cloud 具备：

1. **完整考试状态机**：draft → scanning → grading → reviewing → completed → published → archived
2. **题块级阅卷任务分配**：按题目将阅卷任务分配给不同教师（教育行业标准做法）
3. **质量抽检机制**：随机抽样 + 一致性检查，保障阅卷质量
4. **成绩发布流水线**：发布触发排名计算 + 错题更新 + 事件通知
5. **Agent 工具**：3 个新工具，让 AI 助手能查询和管理阅卷进度

### 2.2 现有阅卷模型

| 模型 | 说明 | 缺什么 |
|------|------|--------|
| GradingTask | AI 阅卷任务（整科级） | 无题块级分配 |
| AIGradingResult | 单道题的 AI 评分结果 | 有 review_status 但无分配关系 |
| TeacherReview | 教师审核记录（approve/override） | 只审 AI 结果，无独立阅卷 |
| Rubric | 评分标准 | 完整 |

**核心缺失**：没有"哪个老师负责改哪些题"的分配机制。当前流程是：AI 全部批改 → 教师逐个审核。缺乏人工阅卷和质量控制。

### 2.3 好分数参考

好分数的考试流程（从 `business-logic.md`）：
- 阅卷任务按题目分配给教师（一个老师改一道或几道题）
- 支持双评（同题两人独立评分）+ 仲裁
- 阅卷完成后需要"发布成绩"操作，发布后学生/家长可见
- 发布后自动生成各类统计报告

**本次取舍**：

## 3. 架构设计

### 3.1 状态机扩展

```
draft ──→ scanning ──→ grading ──→ reviewing ──→ completed ──→ published ──→ archived
  ↑          │                                       │             │
  └──────────┘ (可回退)                              │             │
                                                     │             │
                                              (前置条件检查)   (触发排名+错题)
```

**新增转换规则**：

```python
_VALID_STATUS_TRANSITIONS = {
    "draft": {"scanning"},
    "scanning": {"grading", "draft"},
    "grading": {"reviewing"},
    "reviewing": {"completed"},
    # 新增
    "completed": {"published"},
    "published": {"archived"},
}
```

**转换前置条件**：

| 转换 | 前置条件 | 说明 |
|------|---------|------|
| completed → published | 所有 GradingAssignment status=completed | 所有阅卷任务完成 |
| completed → published | 无 HIGH severity 质量问题未处理 | 质量检查通过 |
| published → archived | 无 | 归档不设门槛 |

**转换后触发**：

| 转换 | 触发动作 | 说明 |
|------|---------|------|
| reviewing → completed | run_full_pipeline()（现有，保持不变） | AI 阅卷流水线（已是 stub） |
| completed → published | publish_service.on_publish() | **新增**：排名计算 + 错题更新 + EventBus emit |
| published → archived | 无 | 标记归档 |

> **与现有 run_full_pipeline 的关系**：reviewing→completed 仍触发 run_full_pipeline()（现有逻辑不变，当前是 stub）。
> completed→published 是**独立的新流程**，在 publish_service 中实现排名和错题逻辑。
> 两者互不影响：completed 是"阅卷完成"，published 是"成绩对外发布"。

#### GradingAssignment（题块级阅卷任务分配）

```python
class GradingAssignment(Base, IdMixin, TimestampMixin):
    __tablename__ = "grading_assignments"

    exam_id: Mapped[str] = mapped_column(String(36), ForeignKey("exams.id"), index=True)
    subject_id: Mapped[str] = mapped_column(String(36), ForeignKey("subjects.id"), index=True)
    question_ids: Mapped[list] = mapped_column(JSON)
        # 分配的题目 ID 列表，如 ["q1-uuid", "q2-uuid"]
    assigned_to: Mapped[str] = mapped_column(String(36), ForeignKey("users.id"), index=True)
    status: Mapped[str] = mapped_column(String(20), default="pending")
        # "pending" → "in_progress" → "completed"
    graded_count: Mapped[int] = mapped_column(default=0)
        # 已批改的答卷数
    total_count: Mapped[int] = mapped_column(default=0)
        # 需要批改的总答卷数（分配时计算）
    started_at: Mapped[Optional[datetime]] = mapped_column(nullable=True)
    completed_at: Mapped[Optional[datetime]] = mapped_column(nullable=True)
    school_id: Mapped[str] = mapped_column(String(36), ForeignKey("schools.id"), index=True)

    # 双评扩展预留（v1 不启用）
    is_second_grading: Mapped[bool] = mapped_column(default=False)
    paired_assignment_id: Mapped[Optional[str]] = mapped_column(
        String(36), ForeignKey("grading_assignments.id"), nullable=True
    )

    __table_args__ = (
        Index("ix_assignment_exam_teacher", "exam_id", "assigned_to"),
    )
```

**状态转换**：
```
pending ──(教师开始批改)──→ in_progress ──(全部批完)──→ completed
```

**设计决策**：
- question_ids 用 JSON 列表而非关联表：一个分配通常 1-5 道题，JSON 更简洁
- graded_count / total_count 用于进度计算，避免每次查询 count
- 双评字段预留但 v1 不使用（is_second_grading 默认 False）

#### GradingQualityCheck（质量抽检）

```python
class GradingQualityCheck(Base, IdMixin, TimestampMixin):
    __tablename__ = "grading_quality_checks"

    exam_id: Mapped[str] = mapped_column(String(36), ForeignKey("exams.id"), index=True)
    subject_id: Mapped[str] = mapped_column(String(36), ForeignKey("subjects.id"), index=True)
    question_id: Mapped[str] = mapped_column(String(36), ForeignKey("questions.id"))
    check_type: Mapped[str] = mapped_column(String(20))
        # "sampling" — 随机抽样复查
        # "consistency" — 同一教师前后一致性
        # "deviation" — AI 与人工评分偏差
    original_result_id: Mapped[Optional[str]] = mapped_column(
        String(36), ForeignKey("ai_grading_results.id"), nullable=True
    )
        # 被抽检的原始结果（AI 结果或人工结果）
    original_grader_id: Mapped[Optional[str]] = mapped_column(
        String(36), ForeignKey("users.id"), nullable=True
    )
        # 原始评分人
    checker_id: Mapped[Optional[str]] = mapped_column(
        String(36), ForeignKey("users.id"), nullable=True
    )
        # 抽检人（复查的教师）
    original_score: Mapped[float] = mapped_column()
    check_score: Mapped[Optional[float]] = mapped_column(nullable=True)
        # 抽检评分（复查后填入）
    deviation: Mapped[Optional[float]] = mapped_column(nullable=True)
        # |check_score - original_score|，审核后自动计算
    severity: Mapped[Optional[str]] = mapped_column(String(10), nullable=True)
        # "low" (偏差≤10%) / "med" (10-20%) / "high" (>20%)
    status: Mapped[str] = mapped_column(String(20), default="pending")
        # "pending" → "reviewed"
    comment: Mapped[Optional[str]] = mapped_column(Text, nullable=True)
    school_id: Mapped[str] = mapped_column(String(36), ForeignKey("schools.id"), index=True)
```

**抽检类型说明**：

| 类型 | 触发方式 | 说明 |
|------|---------|------|
| sampling | 手动触发，按比例随机抽取已批改的答卷 | 最基础的质量控制 |
| consistency | 手动触发，随机抽取同一教师的多份答卷检查评分一致性 | 检测评分标准漂移 |
| deviation | 自动触发，AI 批改和人工批改同一道题，比较偏差 | 校准 AI 评分模型 |

class GradingAssignmentService:
    @staticmethod
    async def auto_assign(
        db, exam_id, subject_id, teacher_ids: list[UUID], school_id
    ) -> list[GradingAssignment]:
        """按题目 round-robin 分配给教师列表"""
        # 1. 查询该科目的所有题目
        questions = await get_questions_by_subject(db, subject_id)
        # 2. 按题目数量均匀分配
        chunks = round_robin_split(questions, len(teacher_ids))
        # 3. 创建分配记录
        assignments = []
        for teacher_id, question_chunk in zip(teacher_ids, chunks):
            # 4. 统计每道题需要批改的答卷数
            total = await count_answers_for_questions(db, exam_id, question_chunk)
            assignment = GradingAssignment(
                exam_id=exam_id, subject_id=subject_id,
                question_ids=[q.id for q in question_chunk],
                assigned_to=teacher_id, total_count=total,
                school_id=school_id,
            )
            assignments.append(assignment)
        db.add_all(assignments)
        await db.flush()
        return assignments

    @staticmethod
    async def assign_block(
        db, exam_id, subject_id, question_ids, teacher_id, school_id, total_count
    ) -> GradingAssignment:
        """手动分配指定题目给指定教师（F-06: total_count 必填）"""
        # N-01: 重复分配检测
        existing = await db.execute(
            select(GradingAssignment).where(
                GradingAssignment.exam_id == exam_id,
                GradingAssignment.subject_id == subject_id,
                GradingAssignment.status != "completed",
            )
        )
        for a in existing.scalars():
            overlap = set(a.question_ids) & set(question_ids)
            if overlap:
                raise ConflictError(f"Questions {overlap} already assigned to {a.assigned_to}")
        assignment = GradingAssignment(
            exam_id=exam_id, subject_id=subject_id,
            question_ids=question_ids, assigned_to=teacher_id,
            total_count=total_count, school_id=school_id,
        )
        db.add(assignment)
        await db.flush()
        return assignment

    @staticmethod
    async def update_progress(db, assignment_id, graded_count) -> GradingAssignment:
        """更新批改进度，达到 total_count 自动转 completed"""
        assignment = await db.get(GradingAssignment, assignment_id)

        if assignment.status == "pending" and graded_count > 0:
            assignment.status = "in_progress"
            assignment.started_at = utc_now()
        if graded_count >= assignment.total_count:
            assignment.status = "completed"
            assignment.completed_at = utc_now()
        return assignment

    async def list_assignments(db, exam_id, *, school_id, assigned_to=None):
        """F-02: school_id 必填，过滤跨校数据"""
        ...

    @staticmethod
    async def get_progress(db, exam_id, *, school_id) -> dict:
        """返回考试阅卷进度汇总（F-02: school_id scope）"""
        assignments = await GradingAssignmentService.list_assignments(db, exam_id, school_id=school_id)

            "completed": sum(1 for a in assignments if a.status == "completed"),

                    "status": a.status,
                    "graded": a.graded_count,
                    "total": a.total_count,
                    "questions": a.question_ids,
                }
                for a in assignments

#### QualityCheckService（`modules/grading/quality_service.py`）

```python
class QualityCheckService:
    @staticmethod
    async def create_sampling_checks(
        db, exam_id, subject_id, rate: float = 0.1, school_id=None
    ) -> list[GradingQualityCheck]:
        """随机抽样已批改结果的 rate 比例"""
        # 1. 查询已完成的评分结果
        results = await get_completed_results(db, exam_id, subject_id)
        # 2. 随机抽取
        import random
        sample_size = max(1, int(len(results) * rate))
        sampled = random.sample(results, min(sample_size, len(results)))
        # 3. 创建抽检记录
        checks = []
        for result in sampled:
            check = GradingQualityCheck(
                exam_id=exam_id, subject_id=subject_id,
                question_id=result.question_id,
                check_type="sampling",
                original_result_id=result.id,
                original_grader_id=result.grader_id,
                original_score=result.score,
                status="pending", school_id=school_id,
            )
            checks.append(check)
        db.add_all(checks)
        await db.flush()
        return checks

    @staticmethod
    async def review_check(
        db, check_id, checker_id, check_score, comment=None
    ) -> GradingQualityCheck:
        """审核抽检：填入复查分数，自动计算偏差和严重度"""
        check = await db.get(GradingQualityCheck, check_id)
        check.checker_id = checker_id
        check.check_score = check_score
        check.deviation = abs(check_score - check.original_score)
        check.comment = comment
        check.status = "reviewed"

        # 计算严重度（基于题目满分的百分比）
        max_score = await get_question_max_score(db, check.question_id)
        if max_score > 0:
            pct = check.deviation / max_score * 100
            if pct <= 10:
                check.severity = "low"
            elif pct <= 20:
                check.severity = "med"
            else:
                check.severity = "high"

        return check

    @staticmethod
    async def get_quality_report(db, exam_id, *, school_id) -> dict:
        """生成质量报告（F-02: school_id scope）"""
        checks = await list_checks(db, exam_id, school_id=school_id)
        reviewed = [c for c in checks if c.status == "reviewed"]
        return {
            "total_checks": len(checks),
            "reviewed": len(reviewed),
            "pending": len(checks) - len(reviewed),
            "avg_deviation": (
                sum(c.deviation for c in reviewed) / max(len(reviewed), 1)
            ),
            "high_severity_count": sum(
                1 for c in reviewed if c.severity == "high"
            ),
            "has_blocking_issues": any(
                c.severity == "high" and c.status == "reviewed"
                for c in checks
            ),
            "by_type": {
                t: {
                    "count": sum(1 for c in reviewed if c.check_type == t),
                    "avg_deviation": (
                        sum(c.deviation for c in reviewed if c.check_type == t)
                        / max(sum(1 for c in reviewed if c.check_type == t), 1)
                    ),
                }
                for t in ["sampling", "consistency", "deviation"]
            },
        }
```

#### ExamPublishService（`modules/exam/publish_service.py`）

```python
class ExamPublishService:
    @staticmethod
    async def publish(db, exam_id, school_id) -> dict:
        """发布考试成绩"""
        exam = await db.get(Exam, exam_id)

        # 前置条件 1: 状态必须是 completed
        if exam.status != "completed":
            raise StateError(f"Cannot publish exam in status '{exam.status}'")

        # 前置条件 2: 所有阅卷任务已完成
        assignments = await list_assignments(db, exam_id)
        incomplete = [a for a in assignments if a.status != "completed"]
        if incomplete:
            raise StateError(
                f"{len(incomplete)} grading assignments not completed"
            )

        # 前置条件 3: 无 HIGH severity 质量问题
        report = await QualityCheckService.get_quality_report(db, exam_id)
        if report["high_severity_count"] > 0:
            raise StateError(
                f"{report['high_severity_count']} high-severity quality issues unresolved"
            )

        # 状态转换
        exam.status = "published"

        # 触发后续动作
        await _calculate_rankings(db, exam_id, school_id)
        await _update_error_books(db, exam_id, school_id)

        # EventBus 通知（现有 events.py 基础设施）
        from edu_cloud.core.events import event_bus
        await event_bus.emit("exam.published", {
            "exam_id": str(exam_id),
            "school_id": str(school_id),
        })

        return {
            "exam_id": str(exam_id),
            "status": "published",
            "rankings_calculated": True,
        }

    @staticmethod
    async def archive(db, exam_id):
        """归档考试"""
        exam = await db.get(Exam, exam_id)
        if exam.status != "published":
            raise StateError(f"Cannot archive exam in status '{exam.status}'")
        exam.status = "archived"

    @staticmethod
    async def _calculate_rankings(db, exam_id, school_id):
        """计算排名（按科目 + 总分）"""
        subjects = await get_exam_subjects(db, exam_id)
        for subject in subjects:
            # 按科目排名
            results = await get_subject_results(db, exam_id, subject.id)
            sorted_results = sorted(results, key=lambda r: r.score, reverse=True)
            for rank, result in enumerate(sorted_results, 1):
                result.rank = rank

        # 总分排名
        total_scores = await get_total_scores(db, exam_id)
        sorted_totals = sorted(total_scores, key=lambda t: t.total, reverse=True)
        for rank, ts in enumerate(sorted_totals, 1):
            ts.rank = rank

    @staticmethod
    async def _update_error_books(db, exam_id, school_id):
        """更新错题本（失分题自动加入）"""
        # 查找失分率 > 50% 的题目+学生
        results = await get_low_score_results(db, exam_id, threshold=0.5)
        for result in results:
            await upsert_error_book_entry(
                db, student_id=result.student_id,
                question_id=result.question_id,
                exam_id=exam_id, school_id=school_id,
                score=result.score, max_score=result.max_score,
            )
```

## 7. 风险与缓解

| 风险 | 级别 | 缓解 |
|------|------|------|
| 排名计算性能（大考试 1000+ 学生） | MED | 使用 SQL ORDER BY + ROW_NUMBER()，不在 Python 排序 |
| 质量检查 high severity 阻塞发布 | LOW | API 返回明确错误信息，指出哪些检查需要处理 |
| auto_assign 题目不整除 | LOW | 余数分配给第一个教师，测试覆盖 |
| 与 Phase 1d 并行 merge 冲突 | LOW | 文件交叉少，仅 import 行级别冲突 |
| Agent 工具格式兼容 | LOW | 暂用现有格式，Phase 1d 统一升级 |
